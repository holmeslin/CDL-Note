## 2.1 The Value of Data

### 2.1.1 How data creates value

**資料的重要性**：

-   **創新和差異化**：資料是推動創新和差異化的重要因素，是釋放人工智能價值的關鍵。
-   **AI 驅動的業務洞察**：資料為 AI 驅動的業務洞察提供動力，幫助公司做出更好的實時決策，是公司構建和運行應用程序的基礎。

**資料生成和挑戰**：

-   **資料量增長**：每天生成更多資料，資料到達的複雜性和速度正在改變業務環境。
-   **非結構化資料**：最有價值的洞察往往隱藏在非結構化資料點中，需要合適的工具、技能和策略來提取。

**資料捕獲與存儲**：

-   **易於捕獲的資料**：如財務資料，可以在資料庫和電子表格中找到。
-   **難於捕獲的資料**：如客戶在社交媒體上的互動，需要新的存儲和分析方法來獲得洞察。

**機器學習與人工智能**：

-   **資料洞察**：ML 和 AI 幫助組織從過去和現在的資料中生成洞察，並以新的方式感知、預測、推薦和分類資料。
-   **個性化體驗**：如在線零售商使用 ML 攝取實時客戶行為資料，提供個性化建議，提升用戶體驗。

**資料價值實現的挑戰**：

-   **資料障礙**：一些組織難以消除與資料之間的障礙，無法實現具體和可衡量的價值。
-   **資料價值差距**：根據埃森哲報告，68%的組織表示無法從資料中實現具體價值。

**智能資料雲**：

-   **價值創造**：智能資料雲是釋放更多業務價值的關鍵，幫助組織縮小資料價值差距，支持價值創造。

這些要點總結了資料在創新和差異化中的重要性、資料生成的挑戰、機器學習和人工智能在資料洞察中的作用，以及智能資料雲在價值創造中的關鍵角色。

### 2.1.2 Unlocking business value from data

**釋放資料價值的重要性**：

-   **數字化轉型核心**：釋放資料價值是數字化轉型的核心。
-   **結合資料類型**：為生成洞察，可能需要結合不同類型的資料。

**資料類型**：

1. **結構化資料**：

    - **定義**：高度組織和明確定義，通常存儲在表格或資料庫中，易於分析。
    - **示例**：CRM 工具中的客戶行為資料。

2. **半結構化資料**：

    - **定義**：介於結構化和非結構化資料之間，有層次結構但無特定順序。
    - **示例**：電子郵件、HTML、JSON 和 XML 文件。

3. **非結構化資料**：
    - **定義**：沒有預定義資料模型或未按預定義方式組織的信息。
    - **示例**：文本資料（文檔、演示文稿、社交媒體帖子）、資料文件（圖像、音頻文件、視頻）、基礎設施活動和性能資料（服務器日誌、IoT 傳感器資料）。

**非結構化資料的挑戰與機會**：

-   **分析困難**：歷史上非結構化資料難以分析，平均來說，只有不到 1% 被分析或使用。
-   **資料比例**：非結構化資料佔所有新企業資料的 80% 到 90%。

**雲技術的影響**：

-   **工具可用性**：雲技術使得以前昂貴且複雜的工具變得可用，幫助企業利用非結構化資料。
-   **機器學習和 API**：使用機器學習發現趨勢或使用 API 提取資料結構，如 Google Cloud 的 Vision API。

**資料價值的解鎖**：

-   **結構化和非結構化資料**：雲技術釋放了結構化資料和以前未開發的非結構化資料的價值。
-   **實現可能性**：了解不同類型資料可以幫助組織確定資料解決方案的可能性。

這些要點總結了釋放資料價值在數字化轉型中的重要性、不同類型的資料及其挑戰和機會，以及雲技術如何幫助企業從資料中獲取價值。

### 2.1.3 Data management concepts

**現代企業資料管理**：

-   **需求**：組織需要現代化的資料管理方法來處理大量生成的資料。
-   **選項**：包括資料庫、資料倉庫和資料湖。

**資料庫**：

-   **定義**：有組織的資料集合，存儲在表中，可以電子訪問。
-   **類型**：
    -   **關係型資料庫**：使用表格格式，具有明確定義的模式，使用 SQL 查詢，適合處理大量結構化資料。
    -   **非關係型資料庫（NoSQL）**：結構較少，使用靈活的資料模型，適合存儲頻繁變化或多樣化資料。
-   **示例產品**：Google Cloud 的 Cloud SQL、Spanner（關係型資料庫），Bigtable（非關係型資料庫）。

**資料倉庫**：

-   **定義**：存儲和分析來自多個來源的結構化和半結構化資料的企業系統。
-   **功能**：適合即時分析和自定義報告，可以存儲當前和歷史資料，支持商業智能。
-   **示例產品**：Google Cloud 的 BigQuery。

**資料湖**：

-   **定義**：攝取、存儲、探索、處理和分析任何類型或數量原始資料的存儲庫。
-   **功能**：存儲不同類型的資料，無需大量預處理，防止資料污染或偏見，並且可以豐富資料。
-   **示例產品**：
    -   **結構化資料**：Cloud SQL、Spanner、BigQuery。
    -   **半結構化資料**：Datastore、Bigtable。
    -   **非結構化資料**：Cloud Storage。

**資料倉庫與資料湖的互補性**：

-   **角色**：資料倉庫適合業務智能分析，資料湖適合資料工程師和資料科學家進行探索和試驗。
-   **用途**：資料倉庫用戶更接近業務，專注於從資料中獲取洞察；資料湖用戶更接近原始資料，尋找答案和提出問題。

**資料驅動決策的重要性**：

-   **資料民主化**：使用戶能夠更深入地了解業務情況。
-   **競爭優勢**：組織需要 360 度的實時業務視圖來獲得競爭優勢。

這些要點總結了現代企業資料管理的必要性和選項，並詳細介紹了資料庫、資料倉庫和資料湖的定義、功能及其在資料驅動決策中的關鍵角色。

### 2.1.4 The role of data in digital transformation

**資料訪問和類型**：

-   **前所未有的資料訪問**：組織對資料的訪問能力前所未有。
-   **資料類型**：
    -   **第一方資料**：企業從客戶或受眾的交易和互動中收集的專有資料。
    -   **第二方資料**：來自合作夥伴或供應鏈中的其他企業的第一方資料。
    -   **第三方資料**：由不直接與組織的客戶或業務互動的組織收集和管理的資料，如政府、非營利組織、學術來源或行業報告。

**資料的應用**：

-   **內部和外部資料結合**：使用外部資料可以通過提供新的背景和見解大大提高資料的價值。
-   **第三方資料的獲取**：通常在資料市場或交易所共享或購買，例如 Google Cloud Marketplace。

**案例分析：航空公司**：

-   **問題**：廉價航空公司需要準確估算機上餐食數量，以避免浪費或售罄。
-   **解決方案**：通過資料分析重新設想解決方案。
    -   **初步因素**：飛機大小和乘客數量。
    -   **深入分析**：目的地、航班時間和航班連接。
    -   **洞察發現**：例如，往返印度的航班需要多出 73%的素食餐。
-   **結果**：提高了餐食預測的準確性，改善了客戶體驗並提高了餐飲服務的盈利能力。

**資料的力量**：

-   **雲技術的作用**：雲技術可以通過重新設想資料釋放新價值。
-   **普遍適用**：無論在公司中的哪個位置，都可以使用資料來解決挑戰。

這些要點總結了組織如何利用不同類型的資料來獲取洞察，並通過實際案例展示了資料和雲技術在業務決策和運營中的重要性。

### 2.1.5 The data value chain

**資料價值鏈的概念**：

-   **定義**：資料價值鏈將資料處理置於更廣泛的背景中，類似於工廠中的裝配線，逐步為資料添加價值，最終轉化為可行動的見解。

**資料價值鏈的步驟**：

1. **資料生成（Data Genesis）**：

    - **定義**：資料的初始創建，例如網站點擊、刷卡或物聯網設備的傳感器記錄。
    - **特點**：這是價值鏈的原材料。

2. **資料收集（Data Collection）**：

    - **定義**：通過攝取將初始資料單位帶到新的系統。
    - **需求變化**：根據原始資料的量、速度和種類以及分析速度而不同。

3. **資料處理（Data Processing）**：

    - **定義**：將收集到的原始資料轉換為準備得出見解的形式。
    - **特點**：可能需要調整資料，例如合併不同資料集，可以是單階段或多階段操作。

4. **資料存儲（Data Storage）**：

    - **定義**：資料落地、可找到並準備進行分析和操作的地方。
    - **存儲選項**：NoSQL（快速讀寫）、資料倉庫（快速訪問分析）、對象存儲（非結構化資料）等。

5. **資料分析（Data Analysis）**：

    - **定義**：為業務導向的行動提供方向，將處理階段的輸入組裝成最終產品。
    - **目的**：提供見解以指導業務決策。

6. **資料激活（Data Activation）**：
    - **定義**：將分析結果推送給相關業務程序和決策者，以便採取行動並完成價值鏈。
    - **常見激活點**：自動決策應用程序和業務智能儀表板。

**資料價值鏈的靈活性**：

-   **變化性**：隨著技術進步、新輸入可用、勞動力發展或所需輸出變化，最佳價值鏈也會變化。
-   **核心原則**：利用原始資料來執行對業務有益的行動。

這些要點總結了資料價值鏈的概念和步驟，強調了如何將原始資料轉化為可行的業務見解，並適應不斷變化的需求和技術環境。

### 2.1.6 Data governance

**資料治理的背景和重要性**：

-   **資料增長**：過去十年中，資料生成量呈指數級增長，雲技術降低了收集、存儲和分析資料的成本。
-   **挑戰**：組織需要在確保資料安全和防止未經授權使用的同時，使資料民主化並嵌入每個決策中。
-   **資料治理的作用**：設定內部標準和遵守外部標準，確保資料在其全生命周期內對所有利益相關者都可用。

**資料治理的定義和目標**：

-   **定義**：資料治理涉及設定資料政策，管理資料的收集、存儲、處理和處置。
-   **目標**：確保資料的高質量和安全共享，支持業務決策，並符合監管標準和合規需求。

**資料治理的好處**：

1. **提高資料價值**：實施資料治理過程，確保資料質量，並提供平台使資料更易於安全共享。
2. **改進決策**：用戶能獲取所需資料以服務客戶、改進產品和服務並抓住新收入機會。
3. **改進成本控制**：消除資料重複，避免過度購買和維護昂貴的硬件。
4. **增強監管合規**：嚴格的資料治理實踐有助於避免不合規風險並預見新法規。
5. **贏得信任**：通過符合可審計的內外部資料政策，組織贏得客戶和合作夥伴的信任。
6. **管理風險**：減少敏感資料暴露給未授權人員或系統的風險，防止安全漏洞。
7. **資料民主化**：強有力的資料治理確保正確人員獲取正確資料，不會對組織產生負面影響。

**資料治理的必要性**：

-   **合規違規風險**：沒有資料治理的組織可能面臨罰款、資料質量差、資料查找困難和 AI 模型訓練不佳等問題。
-   **資料的重要性**：資料成為組織最有價值的資產，每個組織都需要資料治理。

這些要點總結了資料治理的背景、定義、目標及其在業務運營和決策中的重要性，強調了其帶來的多種好處及必要性。
