# 3. Innovating with Google Cloud Artificial Intelligence

## 3.1 AI and ML Fundamentals

### 3.1.1 AI and ML defined

**人工智能（AI）和機器學習（ML）定義**

-   **人工智能（AI）定義:**

    -   是一個廣泛的領域，使用技術來構建能夠模仿與人類智能相關的認知功能的機器和計算機。
    -   功能包括視覺、語言理解與回應、數據分析和建議等。
    -   AI 不是獨立的系統，而是一組技術，旨在讓系統能夠推理、學習和行動，解決複雜問題。

-   **機器學習（ML）定義:**
    -   是 AI 的一個子集，使機器能從數據中學習，而無需明確編程。
    -   依賴於各種模型來分析大量數據，從見解中學習，做出預測和決策。
    -   隨著時間的推移和數據量的增加，機器學習算法和模型會不斷改進性能。

**AI 和 ML 的區別**

-   **人工智能（AI）:**

    -   是涵蓋多種具體方法和算法的總體術語。
    -   包括機器學習、深度學習、機器人學、專家系統和自然語言處理等子領域。

-   **機器學習（ML）:**
    -   是人工智能的一個子領域，專注於讓機器從數據中學習並進行改進。
    -   ML 模型是程序通過算法在訓練數據上學習後的輸出結果。

**生成式 AI**

-   **定義:**

    -   一種能生成新內容（包括文本、圖像、音頻和合成數據）的人工智能。
    -   Google 將生成式 AI 應用於產品，如 Google Workspace，用於自動化不同任務（如生成文檔摘要）。
    -   Google 提供生成式 AI API 等開發工具包，幫助開發者創建定制的產品和服務。

-   **應用:**
    -   對話機器人、內容生成、文檔合成和產品發現等多種應用程序。

### 3.1.2 How AI and ML differ from data analytics and business intelligence AI 和 ML 與數據分析和商業智能的區別

1. **數據分析和商業智能:**

    - 基於歷史數據，用於計算指標或識別趨勢。
    - 主要查看過去發生的事情，例如儀表板和報告。
    - 幫助企業理解過去的表現，但不足以做出未來的決策。

2. **人工智能（AI）和機器學習（ML）:**

    - 提供解鎖預測能力的關鍵。
    - 幫助企業利用數據做出未來的業務決策。
    - AI 是一個廣泛的領域，涵蓋了許多具體的方法和算法，包括 ML。
    - ML 是 AI 的一個子集，讓機器可以從數據中學習並改進性能。

3. **應用示例:**

    - **傳統數據分析:** Maya 領導國際航空公司的業務戰略和運營團隊，使用歷史年度報告來生成儀表板，展示客戶人口分佈和銷售情況等信息。
    - **AI 和 ML:** Maya 可以通過使用 ML 模型，預測每個航班的滿意度和客戶投訴，提前採取措施。
        - 需要大量數據，如每個航班的乘客數量、航班持續時間、客戶滿意度評分、客戶投訴數量等。
        - 了解導致客戶投訴的因素、天氣報告、季節性指標以及客戶投訴的解決時間數據。

4. **ML 的優勢:**
    - 提供動態調整定價或人員分配的能力，甚至可以根據預測調整餐飲安排。
    - 能夠全年對所有航班進行預測性洞察，從而提升航空公司和客戶的價值。
    - 通過正確的數據，使用 ML 模型揭示預測洞察，為企業帶來實際利益。

AI 和 ML 不僅僅是展示過去發生的事情，而是提供預測和決策能力，讓企業能夠更好地應對未來的挑戰並提升競爭力。

### 3.1.3 Problems that ML is suited to solve 機器學習適合解決的問題

**機器學習（ML）可以解決四個常見的業務問題:**

1. **替換或簡化基於規則的系統:**

    - **示例:** Google 搜索。
    - 傳統搜索引擎使用手工編寫的規則來決定顯示哪些運動隊的結果，這在處理大量查詢和用戶時變得非常複雜。
    - ML 模型可以根據用戶點擊的搜索結果進行訓練，以預測搜索結果的排名，提高搜索的精確度和效率。

2. **自動化流程:**

    - **示例:** Ananda Development（泰國的一家房地產開發商）。
    - 傳統的手動檢查過程耗時且易出錯。通過創建一個移動應用程序，檢查員可以口述描述缺陷，應用程序會自動記錄和分類檢查結果。
    - 使用 Google 的語音轉文本 API 和 Cloud Vision API，Ananda Development 優化了檢查過程，節省了檢查時間和人力成本。

3. **理解非結構化數據:**

    - **示例:** Ocado（全球最大的純在線雜貨超市之一）。
    - 傳統的電子郵件處理耗時且導致客戶體驗差。Ocado 利用 ML 處理自然語言，識別客戶情緒和每封郵件的主題，將其自動路由到相關部門，提高了客戶滿意度和保留率。

4. **個性化:**
    - **示例:** YouTube。
    - 使用機器學習提供個性化推薦，提升用戶體驗。當用戶觀看視頻時，YouTube 提供下一個推薦視頻，增加用戶的參與度和滿意度。
    - 許多企業在其網站上使用相同的方法提供個性化的產品推薦，或推薦新內容如音樂和電影。

**重點摘要**

-   **機器學習** 使計算機系統能夠在積累更多數據時不斷調整和改進，從而提高結果的準確性。
-   **ML** 適合解決四個主要的業務問題：替換或簡化基於規則的系統、自動化流程、理解非結構化數據和個性化體驗。
-   **示例** 包括 Google 搜索的結果排名、Ananda Development 的房產檢查過程自動化、Ocado 的電子郵件處理優化和 YouTube 的個性化視頻推薦。
-   **ML 模型** 不是獨立的解決方案，解決複雜的業務挑戰需要結合多種模型。
-   **機器學習的應用** 遍及各行各業，企業可以通過學習相關課程來了解更多應用場景。

### 3.1.4 Why ML requires high-quality data 為什麼機器學習需要高質量數據

**數據質量對機器學習模型的重要性:**

機器學習模型依賴於大量正確且無錯誤的數據來得出預測見解並做出重複決策。如果數據質量低，模型無法做出準確的預測。確保數據質量的六個維度是：完整性、唯一性、及時性、有效性、準確性和一致性。

1. **完整性（Completeness）:**

    - 確保數據集包含所有必要的信息。如果數據不完整，模型無法學習所有必要的模式來做出準確的預測。
    - 例如，訓練依賴於客戶交易數據集的模型，如果缺少交易日期等關鍵信息，模型的準確性會受到影響。

2. **唯一性（Uniqueness）:**

    - 確保數據集中沒有大量重複的數據。如果數據重複過多，模型會被混淆，無法準確識別模式。
    - 例如，訓練識別犬種的模型，如果數據集中大多數都是拉布拉多犬的照片，模型將難以準確識別其他犬種。

3. **及時性（Timeliness）:**

    - 確保數據是最新的，反映當前的狀態。如果數據過時，模型可能基於過時的信息做出預測。
    - 例如，訓練預測股市波動的模型，如果數據已經幾個月了，就無法及時做出當前的預測。

4. **有效性（Validity）:**

    - 確保數據符合預定義的標準和定義，如類型和格式。無效數據會影響模型的預測能力。
    - 例如，日期格式應該是一致的，如標準格式為年、月、日，但數據集中出現 08-12-2019 這樣的日期，則會被視為無效數據。

5. **準確性（Accuracy）:**

    - 確保數據的正確性，如正確的出生日期或銷售單位數量。準確性側重於數據的形式和內容。
    - 例如，在圖像數據集中，有些圖像可能被錯誤地標記為狗，但實際上是貓，這會影響模型的準確性。

6. **一致性（Consistency）:**
    - 確保數據一致，不包含任何矛盾的信息。如果數據不一致，模型無法做出準確的預測。
    - 例如，在客戶信息數據集中，同一個客戶可能在一個地方顯示為 John Smith，在另一個地方顯示為 J.Smith，這會導致數據不一致。

**結論:**

確保數據質量是成功訓練機器學習模型的關鍵。大多數數據質量問題可以通過獲取更多高質量數據來解決，但需要有目的地收集這些數據。提供完整和正確的數據是數據科學家的責任，因為數據是模型觀察世界的唯一鏡頭。

### 3.1.5 The importance of responsible and explainable AI 負責任和可解釋的人工智能的重要性

**人工智能的潛力與負責任使用:**

人工智能（AI）具有顯著的潛力，可以推動醫學進步、理解語言和促進科學發現。然而，要實現這些潛力，關鍵是負責任地使用人工智能。為此，Google 制定了相關原則和最佳實踐，並設立了計劃來合理化這些努力。

**Google AI 原則:**

1. **社會效益:**
    - 人工智能應對社會有益，推動人類進步。
2. **避免偏見:**
    - 避免創造或強化不公平的偏見，確保公平公正。
3. **安全性:**
    - 構建和測試 AI 的安全性，避免對人類和環境造成危害。
4. **責任:**
    - AI 應對人類負責，確保透明和問責。
5. **隱私保護:**
    - 融合隱私設計原則，保障用戶隱私。
6. **科學卓越:**
    - 堅持高標準的科學研究和技術創新。
7. **符合原則的用途:**
    - AI 應用應符合這些原則，避免不道德的使用。

**Google 不設計或部署 AI 的領域:**

-   導致或可能導致整體危害的技術。
-   主要目的是造成或直接促進對人員傷害的武器或技術。
-   收集或使用信息進行監視，違反國際標準的技術。
-   違反國際法和人權原則的技術。

**鼓勵其他組織的負責任 AI 開發:**

Google 鼓勵其他組織制定自己的 AI 原則，促進負責任的 AI 開發。這包括調試和改進機器學習模型的性能，幫助他人理解模型行為，以及檢測和解決數據和模型中的偏見、漂移和其他差距。

**可解釋人工智能（Explainable AI）:**

Google Cloud 提供了一套工具和框架，稱為可解釋人工智能，幫助用戶理解和解釋機器學習模型的預測。這些工具與多個 Google 產品和服務本地集成，確保透明的 AI 開發，提高最終用戶的信任和透明度。

## 3.2 Google Cloud’s AI and ML Solutions

### 3.2.1 BigQuery ML

**BigQuery ML 概述:**

1. **簡化機器學習:**
    - 在大型數據集上進行機器學習通常需要廣泛的編程和 ML 框架知識，這限制了開發人員的範圍，排除了那些了解數據但缺乏機器學習知識和編程專業技能的數據分析師。
2. **BigQuery ML 的目標:**

    - BigQuery ML 旨在實現機器學習的民主化，使數據分析師（主要是數據倉庫用戶）能夠使用現有的商業智能工具和電子表格來構建和運行模型。

3. **使用 SQL 進行模型訓練:**
    - 使用 SQL 語言直接在 BigQuery 中訓練和訪問模型，無需使用 Python 或 Java 編程，這對數據分析師來說更加方便。
4. **減少複雜性與提高生產速度:**

    - BigQuery ML 減少了工具需求和複雜性，因為不需要為基於 Python 的 ML 框架移動和格式化大量數據。同時，提高了生產速度，因為可以直接在 BigQuery 中進行模型訓練。

5. **整合與擴展:**
    - BigQuery ML 與 Google Cloud 的端到端 AI 和 ML 平台 Vertex AI 集成。當 BigQuery ML 模型註冊到 Vertex AI 模型註冊表時，可以部署到端點進行在線預測，進一步擴展了模型的應用範圍。

**總結:**
BigQuery ML 通過允許數據分析師使用熟悉的 SQL 語言直接在數據倉庫中訓練和運行機器學習模型，簡化了機器學習的流程。這不僅減少了工具的複雜性，還提高了模型訓練和部署的速度，同時與 Vertex AI 的整合進一步擴展了其應用和預測能力。

### 3.2.2 Pre-trained APIs

**Google Cloud's 預訓練 API 簡介:**

1. **適用於無自有訓練數據的情況:**

    - 當組織沒有專業的數據科學家但有業務分析師和開發人員時，Google Cloud 的預訓練 API 是理想選擇。
    - 這些 API 是機器學習方法中最快且最省力的選擇，但其可定制性不如其他方法。

2. **快速構建智能應用:**

    - Google Cloud 的預訓練 API 提供用於分析圖像、視頻和文本等常見任務的機器學習模型，幫助開發人員快速構建智能應用程序。
    - API 可以部署在虛擬私有雲、本地或 Google 的公共雲中，無論機器學習經驗水平如何。

3. **應用示例:**

    - 開發者可以使用 Vision API 構建一個用戶提交照片的移動應用程序，該 API 使用 Google 數據自動檢測圖像中的人臉、物體、文本甚至情感。
    - 自然語言 API 可以幫助企業自動處理網站聯繫表單收到的消息，分類文本並決定評論的性質（如投訴、表揚等）。

4. **其他預訓練 API:**

    - **Cloud Translation API:** 將文本從一種語言轉換為另一種語言。
    - **Speech-to-Text API:** 將音頻轉換為文本以進行數據處理。
    - **Text-to-Speech API:** 將文本轉換為高質量的語音音頻。
    - **Video Intelligence API:** 識別視頻中的運動和動作。

5. **高效的模型訓練:**
    - Google 擁有大量的圖像、文本和機器學習研究人員來訓練其預訓練模型，這意味著用戶需要做的工作更少，並且能夠更快地獲得投資回報。

### 3.2.3 AutoML

**Vertex AI 與 AutoML：自定義機器學習模型的解決方案**

1. **自定義數據訓練模型：**

    - 使用自己的數據訓練模型是一種更具定制性的機器學習方法。
    - Vertex AI 提供了一個統一的用戶界面，整合了構建機器學習的 Google Cloud 服務。

2. **AutoML 與 Vertex AI 功能：**

    - AutoML 和 Vertex AI 允許用戶通過圖形用戶界面（GUI）構建和訓練機器學習模型，而無需編寫代碼。
    - AutoML 自動選擇最佳機器學習模型，通過比較不同的模型和調整參數，自動化和快速完成大量手動工作。

3. **減少複雜性與提高生產速度：**

    - BigQuery ML 簡化了機器學習的複雜性，因為需要的工具更少。
    - 無需為基於 Python 的機器學習框架移動和格式化大量數據，大大提高了生產速度。

4. **AutoML 適用場景：**

    - 適合那些希望生成定制機器學習模型但不願花太多時間編寫代碼和試驗數千個模型的企業。
    - 比如汽車製造公司可以用 AutoML 訓練模型來區分好或有缺陷的零件。

5. **AutoML 的圖像識別示例：**

    - 開發人員可以使用 AutoML 視覺實例並用專業數據訓練它，自動化圖像分類模型的訓練。
    - 模型可以進一步優化並直接從雲端部署。

6. **自然語言處理示例：**
    - 如果文本示例不能很好地適應預訓練的自然語言 API 的分類方案，可以使用 AutoML 自然語言，基於自己的專業數據進行訓練和分類。

**總結：**
Vertex AI 和 AutoML 提供了強大的工具，使企業能夠使用自己的數據構建和優化機器學習模型，簡化了模型訓練過程，並提高了模型部署的效率和精度。

### 3.2.5 TensorFlow

**Google Cloud 的 AI 基礎設施與 TensorFlow**

    - 所有機器學習模型都基於 Google Cloud 的 AI 基礎設施構建。

2. **TensorFlow 平台：**

    - TensorFlow 是一個端到端的開源機器學習平台，擁有靈活的工具、庫和社區資源生態系統，使研究人員能夠創新，開發人員可以構建和部署機器學習驅動的應用程序。

3. **開源共享：**

    - TensorFlow 最初為 Google 內部使用而開發，現已開源，讓所有人都能受益。

4. **TPU（張量處理單元）：**

    - TensorFlow 利用了 Google 定制開發的 TPU，用於加速機器學習工作負載，TPU 是領域專用硬件，計算速度比 GPU 提高 200 多倍。
    - 使用完整的雲 TPU pod，結果處理時間從 26 小時縮短到 7.9 分鐘。

5. **集成與應用：**
    - 雲 TPU 已集成到 Google 的各種產品中，這種最先進的硬件和超級計算技術可在 Google Cloud 產品和服務中使用。

**總結：**
Google Cloud 的 AI 基礎設施，特別是 TensorFlow 平台和 TPU，提供了強大的支持，顯著加速了機器學習模型的訓練和部署過程。這些技術的開源和集成，使得研究人員和開發人員能夠更高效地創新和應用機器學習技術。

### 3.2.7 Considerations when selecting Google Cloud AI/ML solutions 選擇 Google Cloud AI/ML 解決方案時的考慮因素

**選擇 Google Cloud AI/ML 解決方案的考量因素**

1. **速度：**

    - 考慮模型投入生產的時間需求。AI 項目通常需要 3 到 36 個月的時間來計劃和實施。預訓練 API 無需模型訓練，節省時間。自定義訓練需要最長時間，因為需要從頭開始構建 ML 模型。

2. **差異化：**

    - 評估模型的獨特性需求。Google Cloud 提供現成的解決方案，如圖像識別和聊天機器人，適合快速部署。Vertex AI 是 Google Cloud 的統一平台，允許 ML 工程師和數據科學家完全控制 ML 工作流程，支持高度定制的 ML 模型。

3. **專業知識：**

    - AI 項目需要數據工程師、數據科學家和機器學習工程師等角色。組織應考慮現有團隊並制定人員策略，包括提升現有員工技能或與外部顧問合作。

4. **建設 AI 解決方案的努力：**
    - 這取決於問題的複雜性、可用數據量和團隊經驗。任何 AI 項目都需要大量時間、努力和專業知識，以對業務運營產生有價值的影響。

**總結：**
選擇適合的 Google Cloud AI/ML 解決方案需要考慮速度、差異化需求、專業知識和建設解決方案所需的努力。不同的解決方案適合不同的業務需求和團隊背景，組織應根據自身情況選擇最合適的方案。
