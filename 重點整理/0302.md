## 3.2 Google Cloud’s AI and ML Solutions

### 3.2.1 BigQuery ML

1. **簡化機器學習:**
    - 在大型資料集上進行機器學習通常需要廣泛的編程和 ML 框架知識，這限制了開發人員的範圍，排除了那些了解資料但缺乏機器學習知識和編程專業技能的資料分析師。
2. **目標:**
    - BigQuery ML 旨在實現機器學習的民主化，使資料分析師（主要是資料倉庫用戶）能夠使用現有的商業智能工具和電子表格來構建和運行模型。
3. **使用 SQL 進行模型訓練:**
    - 使用 SQL 語言直接在 BigQuery 中訓練和訪問模型，無需使用 Python 或 Java 編程，這對資料分析師來說更加方便。
4. **減少複雜性與提高生產速度:**
    - BigQuery ML 減少了工具需求和複雜性，因為不需要為基於 Python 的 ML 框架移動和格式化大量資料。同時，提高了生產速度，因為可以直接在 BigQuery 中進行模型訓練。
5. **整合與擴展:**
    - BigQuery ML 與 Google Cloud 的端到端 AI 和 ML 平台 Vertex AI 集成。當 BigQuery ML 模型註冊到 Vertex AI 模型註冊表時，可以部署到端點進行在線預測，進一步擴展了模型的應用範圍。

**總結:**

BigQuery ML 通過允許資料分析師使用熟悉的 SQL 語言直接在資料倉庫中訓練和運行機器學習模型，簡化了機器學習的流程。這不僅減少了工具的複雜性，還提高了模型訓練和部署的速度，同時與 Vertex AI 的整合進一步擴展了其應用和預測能力。

### 3.2.2 Pre-trained APIs 預訓練 API

1. **適用於無自有訓練資料的情況:**
    - 當組織沒有專業的資料科學家但有業務分析師和開發人員時，Google Cloud 的預訓練 API 是理想選擇。
    - 這些 API 是機器學習方法中最快且最省力的選擇，但其可定制性不如其他方法。
2. **快速構建智能應用:**
    - Google Cloud 的預訓練 API 提供用於分析圖像、影片和文字等常見任務的機器學習模型，幫助開發人員快速構建智能應用程序。
    - API 可以部署在虛擬私有雲( virtual private cloud )、本地( on-premise )或 Google 的公共雲中，無論機器學習經驗水平如何。
3. **應用示例:**
    - 開發者可以使用 Vision API 構建一個用戶提交照片的移動應用程序，該 API 使用 Google 資料自動檢測圖像中的人臉、物體、文字甚至情感。
    - 自然語言 API 可以幫助企業自動處理網站聯繫表單收到的消息，分類文本並決定評論的性質（如投訴、表揚等）。
4. **其他預訓練 API:**

    - **Cloud Translation API:** 將文本從一種語言轉換為另一種語言。
    - **Speech-to-Text API:** 將音頻轉換為文本以進行資料處理。
    - **Text-to-Speech API:** 將文本轉換為高質量的語音音頻。
    - **Video Intelligence API:** 識別視頻中的運動和動作。

5. **高效的模型訓練:**
    - Google 擁有大量的圖像、文本和機器學習研究人員來訓練其預訓練模型，這意味著用戶需要做的工作更少，並且能夠更快地獲得投資回報。

### 3.2.3 AutoML

**Vertex AI 與 AutoML：自定義機器學習模型的解決方案**

1. **自定義資料訓練模型** :

    - 使用自己的資料訓練模型是一種更具定制性的機器學習方法。
    - Vertex AI 提供了一個統一的用戶界面，整合了構建機器學習的 Google Cloud 服務。

2. **AutoML 與 Vertex AI 功能**

    - AutoML 和 Vertex AI 允許用戶通過圖形用戶界面（GUI）構建和訓練機器學習模型，而無需編寫代碼。
    - AutoML 自動選擇最佳機器學習模型，通過比較不同的模型和調整參數，自動化和快速完成大量手動工作。

3. **減少複雜性與提高生產速度：**

    - BigQuery ML 簡化了機器學習的複雜性，因為需要的工具更少。
    - 無需為基於 Python 的機器學習框架移動和格式化大量資料，大大提高了生產速度。

4. **AutoML 適用場景：**

    - 適合那些希望生成定制機器學習模型但不願花太多時間編寫代碼和試驗數千個模型的企業。
    - 比如汽車製造公司可以用 AutoML 訓練模型來區分好或有缺陷的零件。

5. **AutoML 的圖像識別示例：**

    - 開發人員可以使用 AutoML 視覺實例並用專業資料訓練它，自動化圖像分類模型的訓練。
    - 模型可以進一步優化並直接從雲端部署。

6. **自然語言處理示例：**
    - 如果文本示例不能很好地適應預訓練的自然語言 API 的分類方案，可以使用 AutoML 自然語言，基於自己的專業資料進行訓練和分類。

**總結：**
Vertex AI 和 AutoML 提供了強大的工具，使企業能夠使用自己的資料構建和優化機器學習模型，簡化了模型訓練過程，並提高了模型部署的效率和精度。

6. **靈活組合：**

**總結：**
Vertex AI 是一個功能全面的平台，提供了從資料收集到模型部署的全流程支持，適合需要高度定制和專業的機器學習解決方案的企業。它需要專門的團隊來操作，但能夠提供最具創新和差異化的結果。

### 3.2.5 TensorFlow

**Google Cloud 的 AI 基礎設施與 TensorFlow**

1. **基礎設施基礎：**

    - 所有機器學習模型都基於 Google Cloud 的 AI 基礎設施構建。

2. **TensorFlow 平台：**

    - TensorFlow 是一個端到端的開源機器學習平台，擁有靈活的工具、庫和社區資源生態系統，使研究人員能夠創新，開發人員可以構建和部署機器學習驅動的應用程序。

3. **開源共享：**

    - TensorFlow 最初為 Google 內部使用而開發，現已開源，讓所有人都能受益。

4. **TPU（張量處理單元）：**

    - TensorFlow 利用了 Google 定制開發的 TPU，用於加速機器學習工作負載，TPU 是領域專用硬件，計算速度比 GPU 提高 200 多倍。
    - 使用完整的雲 TPU pod，結果處理時間從 26 小時縮短到 7.9 分鐘。

5. **集成與應用：**
    - 雲 TPU 已集成到 Google 的各種產品中，這種最先進的硬件和超級計算技術可在 Google Cloud 產品和服務中使用。

**總結：**
Google Cloud 的 AI 基礎設施，特別是 TensorFlow 平台和 TPU，提供了強大的支持，顯著加速了機器學習模型的訓練和部署過程。這些技術的開源和集成，使得研究人員和開發人員能夠更高效地創新和應用機器學習技術。

### 3.2.6 AI solutions AI 解決方案

**Google Cloud 的完整 AI 解決方案**

1. **概述：**

    - Google Cloud 提供了一組完整的 AI 解決方案，旨在解決特定的業務需求，除了可定制的選項外，這些解決方案更針對性地提供了現成的功能。

4. **Discovery AI**

    - 使用機器學習優化電子商務網站上商品的排序，通過使用歷史資料學習每個頁面的理想產品排序，優化展示產品的方式和內容，提升準確性、相關性和銷售可能性。

5. **雲人才解決方案：**
    - 使用 AI 進行求職和人才獲取，快速將候選人與理想的工作匹配，幫助雇主吸引和轉化更高質量的候選人。

**總結：**
Google Cloud 提供了一系列完整的 AI 解決方案，針對不同的業務需求，如聯絡中心、文檔處理、電子商務和人才獲取等，通過提高效率、個性化服務和優化運營等方式，幫助企業實現更高效的業務運營和決策。

### 3.2.7 Considerations when selecting Google Cloud AI/ML solutions 選擇 Google Cloud AI/ML 解決方案時的考慮因素

**選擇 Google Cloud AI/ML 解決方案的考量因素**

1. **速度( speed )**

    - 考慮模型投入生產的時間需求。AI 項目通常需要 3 到 36 個月的時間來計劃和實施。預訓練 API 無需模型訓練，節省時間。自定義訓練需要最長時間，因為需要從頭開始構建 ML 模型。

2. **差異化( differentiation )**

    - 評估模型的獨特性需求。Google Cloud 提供現成的解決方案，如圖像識別和聊天機器人，適合快速部署。Vertex AI 是 Google Cloud 的統一平台，允許 ML 工程師和資料科學家完全控制 ML 工作流程，支持高度定制的 ML 模型。

3. **專業知識( expertise )**

    - AI 項目需要資料工程師、資料科學家和機器學習工程師等角色。組織應考慮現有團隊並制定人員策略，包括提升現有員工技能或與外部顧問合作。

4. **建設 AI 解決方案的努力( the effort required to build an AI solution )**
    - 這取決於問題的複雜性、可用資料量和團隊經驗。任何 AI 項目都需要大量時間、努力和專業知識，以對業務運營產生有價值的影響。

**總結：**
選擇適合的 Google Cloud AI/ML 解決方案需要考慮速度、差異化需求、專業知識和建設解決方案所需的努力。不同的解決方案適合不同的業務需求和團隊背景，組織應根據自身情況選擇最合適的方案。
